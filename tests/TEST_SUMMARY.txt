# KEY TEST PATTERNS USED
# ======================

1. **Mocking External Dependencies**
   - Mock WhisperModel for transcription
   - Mock NeuralDiarizer for MSDD model
   - Mock SortformerEncLabelModel for Sortformer
   - This allows tests to run without heavy dependencies

2. **Fixture-Based Testing**
   - Reusable fixtures for common test data
   - Temporary directories for file operations
   - Mock models and results
   - Sample audio tensors

3. **Parametrized Testing**
   - process_language_arg with various language codes
   - format_timestamp with different time values
   - Device selection (cpu/cuda)

4. **Edge Case Testing**
   - Empty inputs
   - Boundary conditions (zero, max values)
   - Type conversions (numpy, torch)
   - Missing or None values

5. **Integration Testing**
   - End-to-end diarization pipeline
   - File I/O operations
   - Multi-component interactions
